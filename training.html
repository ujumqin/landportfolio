Learning Objectives

By the end of this module, you will be able to:

  • Reframe feedback in a constructive manner.
  • Understand what to provide feedback about.
  • Deliver feedback in an appropriate timeframe.
  • Recognize the impact of words and tone.
  • Know when to accept, reject, or ask for additional context.

How Should We View Feedback?

Think of feedback like a Software Bug Report.

When you find a bug in code, you don't get angry at the code. You:

  • Triage the issue.
  • Diagnose the root cause.
  • Fix it (or decide if it's "working as intended").
"Something may not be right so triage the issue... and fix it (or don’t if it’s not a true issue)."

The Importance of Words (and Tone)

Small changes in language can change a conflict into a collaboration.

Avoid Absolute Words:
Words like always, never, everyone can trigger defensiveness.
Instead use: often, sometimes, several people.

Avoid Negatively Charged Words:
Words like weakness, problem, mistake.
Instead use: area of improvement, challenge, learning opportunity.

Tone is important so check to see how you are coming off with another person!

Quiz Question 1

Scenario: You notice a teammate is delaying a release.

“You always ask for help too late. We are behind on our milestone because you can’t finish your work on time and the team is suffering.”

Is this an example of good feedback?

a) True
b) False
Correct! This is accusatory. A better approach focuses on flagging blockers sooner so the team can pitch in. For example "I’ve noticed you are getting stuck on the API integration JIRA issue and unfortunately this delay caused us to miss a milestone. I know the API integration is complex, but know you can flag blockers sooner so the team can pitch in. What can we do to make it easier for you to ask for help earlier next time?"

Quiz Question 2

When is a good time to provide feedback?

During a weekly touch base
As close to the situation as possible
When someone complains
After the heat death of the sun (i.e. never)
Correct! The more immediate the feedback, the better. Weekly touch bases are good, but don't save everything for one meeting. Someone complaining does not warrant immediate feedback and more context should be sought.

Quiz Question 3

Why would we want to reframe feedback?

It helps us present problems more objectively
We can choose to not apply the feedback
We can continue to work on them for long periods of time
All of the above
Correct! Reframing helps remove emotion, allows for objective analysis, and helps us decide if the feedback is actionable.

Quiz Question 4

All feedback is relevant and should always be taken to heart.

a) True
b) False
Correct! Not all feedback will be good feedback. Have a conversation to understand the root cause before accepting or rejecting it.

Summary

  • Be Objective: Treat feedback like a bug report.
  • Be Timely: Deliver feedback as soon as possible, provided you have context.
  • Watch Your Tone: Avoid absolutes ("always/never") and accusatory language.
  • Context Matters: Not all feedback is relevant—have a discussion to find the root cause.
